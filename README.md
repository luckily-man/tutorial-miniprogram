# 教程大全 · 微信小程序（云开发共享版）

一个**用户上传并查看图文 + 视频教程**的微信小程序。场景示例：怎么坐地铁、怎么坐飞机、怎么做某道菜……教程由用户上传（文字步骤 + 视频），**所有打开小程序的人都能看到**。

本工程基于**微信云开发（CloudBase）**：教程存云端数据库、视频存云存储，无需自建服务器，即可实现跨设备、多人共享。

---

## 一、功能

- 🏠 **首页**：搜索 + 分类（全部 / 衣 / 食 / 住 / 行 / 生活 / 学习 / 其它）+ 教程卡片列表（来自云端，所有人可见）
- 📖 **详情页**：分步文字说明 + 视频播放（云存储视频免域名校验，真机可播）；管理员/上传者本人可删除
- ➕ **上传页**：填标题 / 选分类 / 写简介 / 动态添加文字步骤 / 选择或拍摄视频（视频自动上传到云存储）
- 👤 **我的**：上传者查看并删除自己上传的教程；管理员查看全部并可删除任意教程
- ⚙️ **管理**：仅管理员可见，生成角色邀请码 / 列出用户并一键改角色

### 权限模型（三种角色）

| 角色 | 查看教程 | 上传教程 | 删除 | 用户管理 |
|---|---|---|---|---|
| 普通用户 `user` | ✅ | ❌ | ❌ | ❌ |
| 上传者 `uploader` | ✅ | ✅ | 仅自己上传的 | ❌ |
| 管理员 `admin` | ✅ | ✅ | 任意教程 | ✅ |

- **首位打开小程序的使用者自动成为管理员**；其余新用户默认 `user`。
- **分配角色不再依赖 openid**（普通用户无需知道自己的 openid）：
  - 管理员在「管理」页点「生成邀请码」得到 6 位码（上传者 / 管理员两种），把码发给对方；
  - 对方在「我的」页「邀请码激活」里粘贴即可**自助升级**身份，7 天内有效、用后失效。
  - 管理员也可在「用户列表」里对已打开过小程序的用户一键改角色。

数据存于微信云开发：数据库集合 `tutorials`（教程）+ `roles`（角色）+ 云存储目录 `tutorials/`（视频）。内置 4 条示例教程（行：坐地铁 / 坐飞机 / 食：番茄炒蛋 / 生活：手冲咖啡），首次启动自动写入云端。

## 二、如何运行（重要：需 3 步前置配置）

### 1. 导入项目
1. 安装 [微信开发者工具](https://developers.weixin.qq.com/miniprogram/dev/devtools/download.html)。
2. 打开开发者工具 →「导入项目」→ 目录选择本文件夹 `wechat-tutorial-miniprogram`。
3. AppID 填你自己的小程序 AppID（本工程已配好 `wxeef4baa9cc564806`，如换了号自行修改 `project.config.json` 的 `appid`）。

### 2. 开通云开发并填入环境 ID
1. 工具顶部点「**云开发**」按钮 → 开通 → 新建一个环境（个人主体有免费额度）。
2. 进入「**环境设置**」复制你的**环境 ID**（形如 `cloud1-xxxxxxxx`）。
3. 打开 `app.js`，把顶部的占位常量替换成你的环境 ID：
   ```js
   // ⚠️ 把下面替换成你自己的「云开发环境 ID」
   const CLOUD_ENV = 'cloud1-xxxxxxxx'   // ← 改成你的
   ```
   > 若未填写，编译时会弹窗提示「未配置云环境」，属正常引导。

### 3. 创建数据库集合并设置权限（关键）
1. 在「云开发」控制台左侧点「**数据库**」→「新建集合」，名称填 **`tutorials`**。
2. 进入该集合 →「**权限设置**」→ 选择 **「所有用户可读，仅创建者可读写」**。
   - ⚠️ 这一步决定「**所有人都能看**」是否成立。默认权限「仅创建者可读写」会导致其他人看不到教程！
3. （可选）「**存储**」→ 确认 `tutorials/` 目录存在；云存储默认「所有用户可读」，视频人人可播。如看不到视频，可在存储权限里设为「所有用户可读」。

### 3.5 新建 roles 集合并部署云函数（权限系统必需）

权限（普通/上传者/管理员）由云端函数校验，因此**必须部署以下 7 个云函数**（`roles` / `invites` 集合会在首次调用时自动创建，也可手动在控制台新建），否则上传/删除/管理/邀请码都会失败。

1. （可选）在「云开发」控制台「数据库」手动新建集合 **`roles`**（权限默认即可，客户端不直接读写它）。`invites` 集合由 `createInvite`/`acceptInvite` 在首次调用时自动创建。
2. 在开发者工具左侧「资源管理器」找到本工程的 `cloudfunctions/` 目录，对其下 **7 个文件夹逐个**右键 →
   **「上传并部署：云端安装依赖（不上传 node_modules）」**：
   - `getRole` —— 启动鉴权，首位使用者自动成为管理员
   - `addTutorial` —— 校验上传权限后写入教程
   - `removeTutorial` —— 校验删除权限（管理员任意 / 上传者仅自己）
   - `setRole` —— 仅管理员修改他人角色
   - `listUsers` —— 仅管理员列出用户
   - `createInvite` —— 仅管理员生成角色邀请码
   - `acceptInvite` —— 用户粘贴邀请码自助升级身份（无需 openid）
   > 部署后可在「云开发 → 云函数」看到它们，点「测试」可验证。

### 4. 编译运行
点击「编译」，模拟器即可看到教程列表；点「预览」扫码即可在手机上查看，**别人手机打开也能看到你上传的教程**。

## 四、工作原理

- **共享**：教程写入云数据库集合 `tutorials`，权限设为「所有用户可读」，因此任何打开小程序的人都能在首页/详情看到。
- **视频**：上传时 `wx.cloud.uploadFile` 存到云存储，返回 `fileID`（`cloud://...`）写入记录；`<video>` 直接用 `fileID` 播放，免域名校验，真机可播。
- **角色与鉴权（双重校验）**：
  - 启动 `getRole` 云函数返回当前用户 `openid` 与 `role`，存入本地与 `globalData`；`roles` 集合无该用户且为空时，首位使用者自动成为 `admin`。
  - 前端按 `role` 控制 UI：普通用户只显示「首页」；上传者/管理员额外显示「上传」「我的」；仅管理员显示「管理」。
  - 写操作走云函数并在**服务端**二次校验：`addTutorial` 仅 `uploader`/`admin` 可写；`removeTutorial` 管理员可删任意、`uploader` 仅删自己（`_openid` 匹配）；`setRole`/`listUsers` 仅 `admin` 可调用；`createInvite` 仅 `admin` 可生成邀请码，`acceptInvite` 任意用户可激活但**只升不降**。普通用户无法绕过直接写库。
- **示例数据**：`app.js` 启动时若云端 `tutorials` 集合为空，自动把 `seed.js` 的 4 条写入（仅一次）。

## 六、回退到「本地存储版」（不需要云开发时）

若暂时不想用云开发，可把 `utils/store.js` 与 `app.js` 改回本地存储实现（历史版本即用 `wx.setStorage`）。本地版教程仅存本机，不共享。
